La Cascada CSS y Especificidad

Entiende cómo CSS decide qué estilos aplicar

1. Orden de Aparición

Cuando dos reglas tienen la misma especificidad, la que aparece última en el CSS gana.

Ejemplo: Última regla gana

Este texto es AZUL porque la última regla gana

/* En el CSS: */
.demo-orden { color: red; }     /* Primera */
.demo-orden { color: green; }   /* Segunda */
.demo-orden { color: blue; }    /* Tercera ← GANA */

Mismo selector, diferentes archivos

Si tienes varios archivos CSS, el orden de los <link> importa. El último archivo cargado tiene prioridad.

<link rel="stylesheet" href="base.css">
<link rel="stylesheet" href="theme.css">  <!-- Este gana -->

2. Especificidad

La especificidad determina qué regla gana cuando hay conflicto. Se calcula sumando puntos según el tipo de selector.

Jerarquía de Especificidad

Elemento p, div, span = 1 punto
Clase / Atributo / Pseudo-clase .card, [type], :hover = 10 puntos
ID #header, #main = 100 puntos
Inline Style style="..." = 1000 puntos

Demostración en vivo

Este texto es VERDE porque #id (100) > .clase (10) > p (1) 

/* Las 3 reglas aplican al mismo elemento: */
p { color: red; }                    /* 0,0,0,1 = 1 punto */
.texto-demo { color: blue; }         /* 0,0,1,0 = 10 puntos */
#demo-especificidad { color: green; } /* 0,1,0,0 = 100 puntos ← GANA */

3. Calculadora de Especificidad

Cómo calcular la especificidad

Cuenta los selectores de cada tipo: (inline, ID, clase, elemento)

Selector Inline ID Clase Elem Total
p 0 0 0 1 1
.card 0 0 1 0 10
#header 0 1 0 0 100
div.card 0 0 1 1 11
ul li a 0 0 0 3 3
.nav .menu .item 0 0 3 0 30
#nav ul.menu li a:hover 0 1 2 3 123
style="color:red" 1 0 0 0 1000

Duelo de Selectores

div.card.active 0,0,2,1 = 21
VS
#sidebar .widget 0,1,1,0 = 110 ✓
html body div.container p.text 0,0,2,4 = 24 ✓
VS
.text 0,0,1,0 = 10

4. La Regla !important

⚠️ !important rompe el flujo natural de la cascada. Úsalo con mucho cuidado.

!important supera toda especificidad

Este texto es ROJO por !important, aunque #id normalmente ganaría 

#demo-important { color: blue; }         /* 100 puntos */
.texto-important { color: red !important; } /* ← GANA por !important */

Cómo sobrescribir un !important

Este texto es VERDE: un !important más específico gana 

.texto-override { color: red !important; }
p.texto-override { color: green !important; } /* ← Más específico + !important */

⚠️ Cuándo usar !important:

  • ✅ Clases utility (como Tailwind: .hidden { display: none !important; })
  • ✅ Sobrescribir estilos de librerías externas que no puedes modificar
  • ❌ NO para "arreglar" problemas de especificidad en tu propio código
  • ❌ NO como primera solución cuando algo "no funciona"

5. Herencia CSS

Algunas propiedades se heredan de padres a hijos automáticamente, otras no.

Propiedades que SÍ se heredan

Padre con color: purple y font-family: Georgia

Hijo: hereda color y font-family automáticamente

Nieto: también hereda (la herencia es recursiva)

Se heredan: color, font-*, text-*, line-height, visibility, cursor, letter-spacing, word-spacing

Propiedades que NO se heredan

Padre con borde, padding y background

Hijo: NO hereda borde, padding ni background

NO se heredan: margin, padding, border, background, width, height, display, position

Valores especiales: inherit, initial, unset

border: inherit

Fuerza la herencia

color: initial

Valor inicial (negro)

all: unset

Resetea todo

6. Resumen: El Algoritmo de la Cascada

Orden de prioridad (de mayor a menor)

  1. 1
    Transiciones CSS Las animaciones activas tienen máxima prioridad
  2. 2
    !important del usuario Estilos del usuario con !important (accesibilidad)
  3. 3
    !important del autor Tu CSS con !important
  4. 4
    Estilos inline style="" en el HTML
  5. 5
    Estilos del autor (por especificidad) Tu CSS, ordenado por ID > Clase > Elemento
  6. 6
    Estilos del navegador User Agent Stylesheet (los estilos por defecto)
